02 / 06

overview

kind とは？

🐳
Kubernetes in Docker
Docker コンテナを Kubernetes ノードとして起動する。インフラ不要でローカルに本物の k8s が動く。
⚡
本番環境に近い挙動
minikube と異なり、マルチノードクラスタも作れる。本番で使う仕組みをそのままローカルで再現できる。
🎓
学習に最適
起動・削除が数秒。壊しても一瞬で作り直せる。Ingress や LoadBalancer の挙動も学べる。
前提条件: Docker Desktop がインストール済み・起動中であること
03 / 06

architecture

クラスタとは？

Kubernetes クラスタ
Control Plane
司令塔 — スケジューリング・状態管理
Node 1
Pod × 2
Node 2
Pod × 2
Node 3
Pod × 1
Control Plane
クラスタ全体を管理する頭脳。
「どのノードで何を動かすか」を決め、状態を常に監視・維持する。
Node
アプリのコンテナ（Pod）が実際に動くマシン。
物理・仮想どちらでも可。
kind では、これらのノードが Docker コンテナとして動く。本物の Kubernetes と同じ仕組みをローカルで再現している。

05 / 06

create cluster

クラスタ作成

Terminal
# クラスタを作成
kind create cluster --name mycluster
Creating cluster "mycluster" ...
✓ Ensuring node image
✓ Preparing nodes
✓ Writing configuration
✓ Starting control-plane
Set kubectl context to "kind-mycluster"
💡 コンテキストとは
「どのクラスタに対して操作するか」の設定。複数クラスタを扱うとき kubectl config use-context で切り替えて使う。

作成後の流れ

1
Docker コンテナとしてノードが起動する
2
kubectl のコンテキストが自動で kind-mycluster に切り替わる
3
kubectl get nodes でクラスタの状態を確認できる
